Nolqu Language Reference

Nolqu is dynamically typed, with a hand-written lexer, recursive-descent parser, bytecode compiler, and stack-based VM. Runtime core in C11, tooling in C++17. Zero external dependencies.

Quick start: nq help lists all built-ins. nq repl starts interactive mode. nq check file.nq parses and compiles without running.

Syntax Basics

Blocks are opened by keywords and closed by end. No curly braces. Newlines are the statement terminator — no semicolons needed. Comments start with #.

# This is a comment
print "Hello"          # Hello
print 10 + 5           # 15
print "A" .. "B"       # AB  (..) is string concat

Variables

Declare with let. Dynamically typed. Reassign without let. Variables declared at the top level are global; inside a block they are local.

let name  = "Alice"
let count = 0
let flag  = true
let empty = nil
count = count + 1   # reassign — no let

Prefix a variable name with _ to suppress unused-variable warnings: let _ignored = some_call()

Operators

Arithmetic: + - * / %. Division and modulo by zero throw a catchable error. String concat: .. (auto-converts numbers and booleans). Logical: and or not (also ! as alias for not).

loop

loop is a while-style loop — runs as long as the condition is truthy. Note: there is no break or continue in v1.0.0.

let i = 1
loop i <= 10
  print i
  i = i + 1
end

Functions

Functions are first-class values — store them in variables, pass as arguments. A function with no return returns nil. Note: closures are not supported in v1.0.0 — functions can only access globals and their own locals.

function add(a, b)
  return a + b
end
print add(3, 4)   # 7

# Recursion
function factorial(n)
  if n <= 1
    return 1
  end
  return n * factorial(n - 1)
end

Strings

Strings are immutable and UTF-8. All operations return new strings.

let s = "Hello, World"
print len(s)                      # 12
print upper(s)                    # HELLO, WORLD
print lower(s)                    # hello, world
print trim("  hi  ")             # hi
print slice(s, 0, 5)             # Hello
print replace(s, "World", "Nolqu") # first only
print index(s, "World")          # 7
print startswith(s, "Hello")    # true
let parts = split(s, ",")         # array

Modules

Path is relative to the working directory where nq is invoked. The .nq extension is added automatically. Importing executes the file in the global scope — all top-level declarations become globals.

import "stdlib/array"   # map  filter  reduce  reverse
import "stdlib/file"    # read_or_default  write_lines  count_lines
import "my_module"      # loads my_module.nq from working dir

Scoping Rules

let at top level → global. let inside a function → local to that function. let inside if, loop, try, or catch → local to that block. Local variables shadow outer ones with the same name.

let x = "global"

function test()
  let x = "local"   # shadows global x
  print x           # local
end

test()
print x             # global

VM Design

Nolqu's pipeline: Lexer → Parser → Compiler → VM. The runtime core (memory, value, chunk, object, table, gc, vm) is C11. Tooling (lexer, parser, compiler, repl) is C++17.

ResourceLimit
Operand stack512 values
Call frames64
try/catch nesting64
Locals per function256
Constants per chunk256
Host-registered natives64

GC is mark-and-sweep, triggered automatically. Threshold starts at 1 MB and doubles after each collection. The VM also includes "Did You Mean?" suggestions for undefined variables using Levenshtein distance.

Embedding in C / C++

Include nolqu.h and link all .c and .cpp files from src/ together with your application.

// Quick start
#include "nolqu.h"

NqState* nq = nq_open();
nq_dofile(nq, "script.nq");
nq_close(nq);
// Pass data in, read results out
nq_setglobal_string(nq, "player_name", "Alice");
nq_setglobal_number(nq, "level", 5);
nq_dofile(nq, "game.nq");
NqValue score = nq_getglobal(nq, "final_score");
// Register a host function
NqValue host_get_time(NqState* nq, int argc, NqValue* argv) {
  return nq_number((double)time(NULL));
}
nq_register(nq, "get_time", host_get_time, 0);

API functions: nq_open nq_close nq_dostring nq_dofile nq_setglobal_* nq_getglobal nq_register nq_gc nq_lasterror. Up to 64 host functions per state. States are not thread-safe.

Grammar (EBNF)

Formal grammar specification for Nolqu v1.0.0.

program     → statement* EOF

statement   → let_stmt | assign_stmt | print_stmt
            | if_stmt | loop_stmt | function_decl
            | return_stmt | import_stmt | try_stmt | expr_stmt

let_stmt    → "let" IDENT "=" expr NEWLINE
if_stmt     → "if" expr NEWLINE statement*
              ( "else" NEWLINE statement* )? "end" NEWLINE
loop_stmt   → "loop" expr NEWLINE statement* "end" NEWLINE
try_stmt    → "try" NEWLINE statement*
              "catch" IDENT NEWLINE statement* "end" NEWLINE

expr        → or_expr
or_expr     → and_expr ( "or" and_expr )*
and_expr    → not_expr ( "and" not_expr )*
not_expr    → "not" not_expr | equality
equality    → comparison ( ( "==" | "!=" ) comparison )*
comparison  → concat ( ( "<" | ">" | "<=" | ">=" ) concat )*
concat      → add ( ".." add )*
add         → mult ( ( "+" | "-" ) mult )*
mult        → unary ( ( "*" | "/" | "%" ) unary )*
unary       → "-" unary | "!" unary | call
call        → primary ( "(" arg_list? ")" | "[" expr "]" )*
primary     → NUMBER | STRING | "true" | "false" | "nil"
            | IDENT | "(" expr ")" | "[" expr_list? "]"

Philosophy

Nolqu was designed around one simple principle: code should read like plain sentences.

The language favors clarity over cleverness. Every feature exists to make programs easier to read and understand.

Architecture

The entire pipeline is hand-written: a custom lexer tokenizes source, a recursive-descent parser builds an AST, and a bytecode compiler emits instructions for a stack-based VM. Mark-and-sweep GC with tunable thresholds. Zero external dependencies.

Platform Support

Compiles and runs on Android (Termux + clang), Ubuntu/Debian/Arch Linux, macOS, and Windows (MinGW-w64 / MSYS2). Same .nq scripts run everywhere without modification.
